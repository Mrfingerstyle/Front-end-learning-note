        Git是开启GitHub的钥匙
        所有软件都要先配置
        
        查看git配置
        git config --global --list

        windows系统
        git config --global user.name 你的英文名
        git config --global user.email 你的邮箱
        git config --global push.default simple
        git config --global core.quotepath false
        git config --global core.editor "code --wait"
        git config --global core.autocrlf input

        上面的用户名和邮箱跟GitHub没有关系 
        可以保持一致 也可以不保持一致

        git只是一个命令而已

        git add 路径 
            此处的路径可以是绝对路径 相对路径 . * 
            选择哪些文件变动需要提交
        
        .gitignore 选择哪些变动不需要提交
            常见不能提交的目录 node_modules .DS_Store .idea .vscode 
            git status 查看文件状态
        
        ls -al 显示所有文件 包含隐藏文件
        
        git commit -m 字符串
        提交 并说明提交理由
        
        改动了index.html  添加了一行代码 
        然后Git add 
        再使用Git commit -v 然后vim弹出一个文件 首行空白 用户在首行填入了提交说明
        编写完成 然后vim 退出插入模式 :wq 退出 
        在命令行显示了刚刚输入的提交说明
        
        git log 查看提交的历史及提交说明
        git commit -v 能帮助用户回顾刚刚做了什么 展示文件的改动内容
            迫使提交的理由更详细一些
            建议新人使用
        git status 查看当前文件状态 有没有改动 未添加或提交 
        git add . 提交当前整个目录
        
        git reset --hard xxxxxx 回退到之前提交的某一个版本
            xxxxxx是提交号的前6位
        输入之后 文件回退到第一次提交时的状态
        再输入 git log 提交信息只有1次
        
        git reflog 不止看当前的提交历史 还要看回滚之前的提交历史
        再次使用git reset 跳转到第二次提交更新的状态 
        再次使用git reset 跳转到第三次提交更新的状态 

        这也解释了之前每次都要commit 是为了后面的回滚方便

        git log只会显示之前的提交
        使用 git reflog可以查看所有提交
        然后用git reset --hard xxxxxx 回退

        运行git reset 之前 确保文件已经提交了 或者忽略
        不能再Git add未提交状态 否则 会丢失改动的内容

        总结
            git branch
            基于当前commit创建一个新的时间线
            我在哪个分支提交 代码就出现在哪个分支
            git checkout 
            用于切换另一个分支
            当前目录有未提交的代码 只要跟另一个分支不冲突 就不需要理会 
            如果冲突 可以使用 git stash 也可以合并冲突
            切分支的时候不影响硬盘上的文件 但是git reset是影响的
        
        git branch 显示当前在哪个分支
        
        场景三 两个版本都要
        
        git merge合并分支
            有可能有冲突 也有可能没有冲突
